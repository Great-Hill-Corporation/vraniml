VRaniML Design Documentation
         Browser Related Classes
         (VRML Execution Model - VEM)

This component provides support for the VRML 2.0 runtime execution model.  In other words, this component supports VRML event ROUTEing.  This component also supports the VRML parser which is a lex/yacc based parser which builds the scene graph from a .wrl file.  The classes making up this component are described below.  For more information click here.

 

Browser class:  This grouping node class serves as the top-level of the scene graph.  The browser is basically the place where all the data is stored.  The browser maintains the VRML clock, insuring that the scene is rendered at a sustained frame rate.  The browser also maintains a list of Traversers supporting such things as multi-pass rendering.

Event class: The event class is used to propagate VRML 2.0 events from node to node along ROUTEs.  The event class helps keep track of what data is to be changed with each event that is propagated.  When an event is initiated (for example from a TimeSensor) it is sent along any ROUTEs using the event class.

Route class: The route class maintains the ROUTEs between nodes.   Each node may ROUTE any of its exposedFields or eventOuts to other nodes exposedFields or eventIns.  The route class holds the data for this connection.

UserMessage class: This class abstracts windows messages such as a mouse movement or keystroke.  It exists mostly to make porting to operating systems other than MS Windows easier.
