VRaniML VRML 2.0 Node Class Library
         VRaniML Design Documentation

This document describes the component design of the library.

The VRaniML SDK is a library of C++ classes intended to help applications support importing, exporting and rendering of VRML 2.0 files.  The current version of the library supports a subset of the VRML 2.0 specification which is detailed here.

A high level block diagram of the SDK is included here:

 

The SDK is composed of five components.  Each component is briefly described here and further defined on the following pages:

Utility - Utility Class Library - A collection of supporting classes and routines for collection classes (lists, arrays, etc.), geometry classes (vertex2, vertex3, rotation, matrix, etc.), and other supporting classes (timer, image, runtime typing support etc.).  This is also called the Virtual Reality Foundation Classes or VRFC for short.
Browser Related Classes - VRML Execution Model - This component supports the VRML 2.0 execution model.   This provides support for the VRML 2.0 ROUTE keyword.  The generation and propagation of VRML eventIn's and eventOut's is handled by this component.  This component also provides support for top level scene graph management as well as PROTO management.  This component contains the VRML 2.0 parser which is implemented using lex/yacc.
VRML97 Node Library - Each VRML 2.0 node has a corresponding VRaniML class.  In effect, these classes simply provide access to the data fields of each node type.  This component provides one class for each of the 54 VRML 2.0 nodes.
Solid Modeling library - All VRaniML geometry nodes (for example IndexedFaceSet or Extrusion) use the solid modeling library as their underlying data representation.   This data representation allows for a number of very useful functions, including future support for such advanced features as automatic level of detail generation and progressive mesh (that is progressive transmission) of geometry over the internet.   Other possible future features might include interactive splitting planes and Boolean operations.
Traversers - This component contains classes that "walk" the scene graph at the applications direction to "render" the scene.  There are a number of Traversers in this component including OpenGLTraveser which renders the scene graph to an OpenGL window, WriteTraverser which "renders" the scene graph to a file (export) and ValidateTraverser which walks the scene graph looking for and reporting on bad data.
