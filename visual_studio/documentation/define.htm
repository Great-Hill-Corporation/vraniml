Utility Class Library Reference Manual
        Defined Symbols

This part of the manual documents a number of #defined values found in the Utility Class Library.  These macros are used mostly for use in implementing runtime typing and dynamic object creation.  These macros are very similar to the corresponding MFC macros.

DECLARE_NODE(className);

This macro allows you to enable runtime typing and dynamic creation for any vrBaseNode derived class.  The macro is added to the class declaration in the same way that the DECLARE_DYNCREATE macro from the MFC.  This macro allows you to use the GETRUNTIME_CLASS and IsKindOf routines in your code.   Also, a static global function called Create[className]* is provided to dynamically create objects of this type.  If you add this macro to the class declaration you must also add the IMPLEMENT_NODE macro in the class's .cpp file.

in className:
    The name of the class (not quoted).

* - for example CreateBox if className equals Box.

IMPLEMENT_NODE(className, parentName, version);

This macro should be included in the implementation file (i.e. .cpp file) for any class that includes the DECLARE_NODE macro.  It corresponds to the IMPLEMENT_DYNCREATE macro in the MFC.

in className:
    The name of the class (not quoted).
in parentName:
    The name of the class (not quoted) this class is derived from.
in version:
    The version of this class (currently unused).

GETRUNTIME_CLASS(className);

This macro can be used on any class that includes the DECLARE_RUNTIME or DECLARE_NODE macros.  The macro returns a vrPrototype class instance that describes information about the requested class type. 

in className:
   The name of the class (not quoted).

vrDELETE(node);

This macro may be used by any vrBaseNode derived class to insure that the node is properly deleted.  The macro decrements the reference count on this node and only deletes the node if the reference count goes to zero.

in node:
   The node to be dereferenced and possibly deleted.
